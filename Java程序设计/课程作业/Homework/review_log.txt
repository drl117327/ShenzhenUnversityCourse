// 案例 1: Buggy Code
public class BuggyCode {
    public static int findMax(int[] arr) {
        if (arr == null || arr.length == 0) {
            return -1;
        }
        int max = arr[0];
        for (int i = 0; i <= arr.length; i++) {
            if (arr[i] > max) {
                max = arr[i];
            }
        }
        return max;
    }
}
```

## 发现的问题

### 1. **数组越界异常（严重缺陷）**
- `for`循环条件使用了 `i <= arr.length`，这会导致数组索引越界
- 数组的有效索引范围是 `0` 到 `arr.length - 1`
- 当 `i = arr.length` 时，`arr[i]` 会抛出 `ArrayIndexOutOfBoundsException`

### 2. **错误的返回值设计**
- 使用 `-1` 作为空数组或null的返回值不合理
- 如果数组中实际的最大值就是 `-1`，调用者无法区分是真正的最大值还是错误状态

- 变量名：首字母小写的驼峰命名法（camelCase）

- 使用适当的缩进（通常为4个空格）

- 合理使用换行提高可读性
- 采用一致的大括号风格

### 3. 优化设计
- 根据实际需求确定方法的返回类型
- 添加必要的Javadoc注释

 * 数值处理器类，提供数值转换相关的业务逻辑

 * @author Developer
 * @version 1.0

public class ValueProcessor {
    
    /**
     * 根据输入值执行处理逻辑
     * 当输入值大于10时，返回其两倍值；否则返回原值
     *
     * @param inputValue 输入的整数值，应为非负数
     * @return 处理后的结果值
     */
    public int processValue(int inputValue) {
        int result;
        
        if (inputValue > 10) {
            result = inputValue * 2;
        } else {
            result = inputValue;
        }
        
        return result;
    }
}
```

}
